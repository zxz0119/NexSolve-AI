# NexSolve AI

> **Bridging Real-World Pain Points with AI Solutions.** > **连接真实世界的痛点与 AI 解决方案。**

---

## 📖 Introduction / 项目简介

NexSolve AI is a decentralized platform built on GitHub. We aim to collect real-world "pain points" from various industries and solve them using AI-native programming.
NexSolve AI 是一个建立在 GitHub 上的去中心化平台。我们旨在收集各行各业的真实“痛点”，并利用 AI 原生编程技术提供解决方案。

- **For Users:** Submit your problems (e.g., "Manual data entry is too slow").
- **For AI Developers:** Claim tasks, build AI solutions, and showcase your skills.
- **For Everyone:** All core solutions contribute to the open-source ecosystem.

---

## 🛠 How It Works / 使用指南

### 1. Submit a Pain Point / 提交需求
Go to the **Issues** tab and click **New Issue**. Choose the "Pain Point" template. Fill in the details about your industry and the specific problem.
点击 **Issues** 标签，选择 **New Issue**，使用“痛点需求”模板填写详情。

### 2. Dev Response / 开发者接单
Developers can browse issues. If interested, reply with "I'm working on this" and add the `in-progress` label.
开发者浏览 Issue，有意向者回复“正在处理”并挂上 `in-progress` 标签。

### 3. Solution & Reward / 成果与报酬
- Developers post the solution link/code in the Issue.
- Rewards/Payments are negotiated directly between the user and the dev via private contact (e.g., Email, WeChat).
- 开发者在 Issue 中回复成果链接或代码。
- 报酬由需求方与开发者自行私下协商。

---

## 🏷 Label Rules / 标签规则

- `pain-point`: New requirement submitted. (新提交的需求)
- `in-progress`: A developer is working on it. (开发中)
- `solved`: A verified solution has been provided. (已解决)
- `bounty-open`: The user is willing to pay for this. (有偿求助)
